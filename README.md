# Bridge Studio

브라우저 안에서 사장교를 직접 다듬고, 시점을 바꿔 보며, 장면까지 함께 설계해 볼 수 있도록 만든 파라메트릭 3D 프로젝트입니다.  
단순히 다리 한 덩어리를 띄우는 데서 끝내지 않고, 주탑, 케이블, 박스거더, 접속부, 수변 구조물, 항로 표지까지 한 장면 안에서 읽히도록 구성했습니다.

![Bridge Studio 대표 화면](docs/assets/bridge-studio-hero.png)

## 프로젝트 개요

`Bridge Studio`는 React와 Three.js 기반으로 만든 브라우저 3D 교량 시각화 프로젝트입니다.  
사용자는 프리셋과 파라미터를 바꿔 가며 사장교의 비례를 조정할 수 있고, 전체화면 뷰포트 안에서 장면을 직접 탐색할 수 있습니다.

이 프로젝트에서 가장 중요하게 본 건 두 가지였습니다.

- 브라우저에서도 충분히 몰입감 있는 교량 장면을 만들 것
- “설계값을 바꾸면 형상이 바로 반응하는” 파라메트릭 경험을 줄 것

## 이런 점을 보여주고 싶었습니다

### 1. 브라우저 안에서 돌아가는 교량 전용 3D 스튜디오

일반적인 웹 페이지가 아니라, 장면 자체가 주인공이 되는 형태로 만들었습니다.  
진입 화면에서 바로 3D 엔진을 띄우지 않고, `Enter Studio` 이후에만 scene을 부팅해 첫 로딩도 다듬었습니다.

### 2. 단순 박스 모델이 아니라, 구조가 읽히는 교량

사장교의 핵심 요소가 그냥 “있다” 수준이 아니라, 어느 정도 구조적으로 읽히는 방향으로 디테일을 쌓았습니다.

- H형 주탑
- 양측 케이블면
- 박스거더 단면
- 난간, 점검통로, 받침, 교대, 접근부
- 수변 접속 구조와 항만형 주변 요소

### 3. 뷰어가 아니라, 직접 만지는 장면

카메라 프리셋만 제공하는 정적 쇼케이스가 아니라, 사용자가 장면을 직접 돌아다니고 저장할 수 있게 만들었습니다.

- Orbit 기반 회전/줌
- `WASD` 이동
- 현재 뷰 PNG 저장
- URL 공유
- 마지막 세션 복원
- 로컬 프리셋 저장

## 주요 기능

- 사장교 파라미터 조절
  - 메인 스팬, 상판 높이, 상판 폭, 주탑 높이, 케이블 수, 케이블 기울기 조절
- 프리셋 기반 빠른 탐색
  - `Compact`, `Balanced`, `Monumental`
- 카메라 프리셋
  - `Hero`, `Front`, `Side`
- 장면 저장/공유
  - URL snapshot 복원
  - last session 자동 복원
  - 로컬 preset 저장/불러오기
- 전체화면 3D 인터랙션
  - 마우스 Orbit
  - 휠 줌
  - `WASD` 카메라 이동
- 이미지 출력
  - 현재 카메라 기준 PNG export

## 구현 포인트

### 파라메트릭 생성

교량은 정적인 모델 파일을 불러오는 방식이 아니라, 입력 파라미터를 바탕으로 형상을 계산해서 생성합니다.  
상판, 주탑, 케이블, 교각, 접속 구조물, shoreline 배치가 서로 다른 기준을 쓰지 않도록 `guides`를 중심으로 정리했습니다.

### 장면 연출

장면은 “교량만 떠 있는 상태”를 피하려고 계속 보강했습니다.

- 교대와 접근도로
- 제방, 호안, 작업 마당
- 항만 apron, bollard, fender wall
- 부표와 beacon 같은 항로 표지
- 황혼 톤의 하늘, 수면, 안개

결과적으로 모델 단독 뷰보다는, 실제 현장 위에 놓인 구조물처럼 보이게 만드는 데 집중했습니다.

### 성능 최적화

처음부터 모든 걸 한 번에 올리는 대신, 진입 경험과 런타임 비용을 나눠서 다뤘습니다.

- Enter gate 이후에만 scene boot
- core scene / scenic layer 분리 로딩
- 반복 구조물 instancing
- 차량/대기 레이어 경량화

웹 3D는 “보이기만 하면 끝”이 아니라, 장면이 커질수록 로딩과 draw call이 바로 문제로 이어지기 때문에 이 부분을 꽤 신경 써서 정리했습니다.

## 프로젝트에서 신경 쓴 부분

- 구조물 형상이 파라미터에 따라 자연스럽게 바뀌는지
- 케이블, 주탑, 상판, 차량, 접속부가 서로 다른 좌표계로 어긋나지 않는지
- 전체화면 3D 경험이 답답하지 않도록 카메라 조작을 다듬는지
- 로딩 전에 scene chunk를 불필요하게 당기지 않는지
- 단순 데모가 아니라, 포트폴리오로 보여도 되는 화면 퀄리티가 나오는지

## 아쉬운 점과 다음 단계

아직 더 다듬고 싶은 부분도 분명히 있습니다.

- `three-core` 번들은 여전히 큰 편이라 추가 최적화 여지가 있습니다.
- 수변 구조물과 도시 배경은 지금도 절차적 저폴리 중심이라, 더 밀도 있게 발전시킬 수 있습니다.
- 교량 타입을 하나 더 늘리면 스튜디오 성격이 더 강해질 수 있습니다.

그래도 현재 버전은 “브라우저에서 이 정도 밀도의 교량 장면을 직접 조작할 수 있다”는 점을 분명하게 보여주는 결과물이라고 생각합니다.
